Getting started with psdness

library(psdness)

The question

A correlation matrix is only valid if it is positive semidefinite (PSD). Reported matrices in papers, though, are rounded — you see 0.62, not the real number behind it. So the honest question is not “is this exact matrix PSD?” but

given that every off-diagonal was rounded to d decimals, does any PSD matrix exist inside the rounding box around the reported values?

psdness answers that box question. Writing delta = 0.5 * 10^(-d), the box is X_ii = 1, and for i != j, X_ij in [R_ij - delta, R_ij + delta] intersected with [-1, 1]. The verdict is one of:

  • impossible — no PSD matrix fits the box, so the reported matrix cannot be a genuine correlation matrix (a rigorous certificate; this is the payload);
  • possible — a PSD matrix in the box is exhibited (or none was ruled out);
  • undecided — precision-limited; the box grazes the PSD boundary.

Checking a matrix

A genuine correlation matrix, rounded, comes back possible:

R_ok <- matrix(c(1,    0.30, 0.20,
                 0.30, 1,    0.25,
                 0.20, 0.25, 1), 3, 3)
check_corr_psd(R_ok, decimals = 2)
#> <corr_psd_check>
#>   verdict : POSSIBLE    (not shown impossible)
#>   tier    : witness
#>   p       : 3      delta : 0.005
#>   margin  : B_upper = 0.699  (>= 0: no impossibility certificate)
#>   evidence: the reported matrix itself is PSD

An inconsistent one comes back impossible. The classic case is a “frustrated triangle”: variables 1 and 2 are both strongly correlated with 3, yet reported as strongly negatively correlated with each other.

R_bad <- matrix(c(1,    0.9,  0.9,
                  0.9,  1,   -0.9,
                  0.9, -0.9,  1), 3, 3)
res <- check_corr_psd(R_bad, decimals = 2)
res
#> <corr_psd_check>
#>   verdict : IMPOSSIBLE  (no PSD matrix fits the rounding box)
#>   tier    : witness
#>   p       : 3      delta : 0.005
#>   margin  : B = -0.79  (B_upper = -0.79 < 0)
#>   witness : v = (0.5774, -0.5774, -0.5774) 
#>   certificate: v'Rv + delta*(||v||_1^2 - ||v||_2^2) = -0.79 < 0

The certificate and the rigor guarantee

An impossible verdict carries a witness vector v: a direction along which no in-box matrix has a nonnegative quadratic form, so none can be PSD. Extract it with certificate():

certificate(res)
#> $witness
#> [1]  0.5774 -0.5774 -0.5774
#> 
#> $margin
#> [1] -0.79
#> 
#> $b_upper
#> [1] -0.79
#> 
#> $verdict
#> [1] "impossible"
#> 
#> $tier
#> [1] "witness"

The guarantee is that the verdict is floating-point-safe: the witness quantity v'Rv + delta * (||v||_1^2 - ||v||_2^2) is evaluated with a rigorous a-priori error bound (Higham 2002), so impossible never rests on a solver tolerance. A corollary is soundness in the other direction: because the bound is a sound over-estimate, a box that contains any PSD matrix can never be reported impossible. In particular a real correlation matrix rounded to d decimals is never a false positive.

Screening a corpus

check_corr_psd_batch() runs the check over a list of matrices and returns a tidy table — useful for screening many reported matrices at once.

mats <- list(ok = R_ok, frustrated_triangle = R_bad, identity = diag(4))
check_corr_psd_batch(mats, decimals = 2, quiet = TRUE)
#> # A tibble: 3 × 8
#>   id                  verdict    tier    margin b_upper delta     p message

Localizing the fault

When a matrix is impossible, localize_psd_fault() asks where the problem is and how severe it is. Non-PSDness is a global property, so a unique culprit is often under-identified; the tool reports an honest localization class rather than inventing one. The verdict is one of cell, cell_tentative, triad, variable, joint, diffuse, or none.

A frustrated triangle is a triad

Fixing any one of the three correlations would resolve the violation, so the fault is not separable to a single cell:

localize_psd_fault(R_bad, decimals = 2)
#> <psd_fault>
#>   Impossible given rounding (delta = 0.005).
#>   Verdict: TRIAD - Attributable to an inconsistent triangle (three cells) 
#>     cells: (1,2), (1,3), (2,3) 
#>   Structural: Substantive: no correlation matrix lies within rounding of the reported values (smallest correction is ~79x the rounding step), so it is not explained by rounding of a structural dependency. 
#>   Severity: largest single edit needed 0.395 (egregious, rounding cannot excuse it; delta = 0.005);
#>             total mass (Frobenius) 0.684; best achievable lambda_min -0.79.
#>   - Not separable: any one of the three edits would resolve the violation. 
#>   - R^2 localizer: variable(s) {1, 2, 3} are over-determined by the others (R^2 > 100%) -- conditional on the other cells being correct.

A single bad cell

Here one transcription-style error makes a 5-variable matrix impossible. The tool localizes it to the offending cell, states the interval that cell should have occupied, and how far beyond rounding it must move:

R_cell <- matrix(c( 1,    0.56,  0.68,  0.05,  0.82,
                    0.56, 1,    -0.26, -0.31, -0.75,
                    0.68, -0.26, 1,    -0.18,  0.50,
                    0.05, -0.31, -0.18, 1,     0.27,
                    0.82, -0.75, 0.50,  0.27,  1), 5, 5, byrow = TRUE)
localize_psd_fault(R_cell, decimals = 2)
#> <psd_fault>
#>   Impossible given rounding (delta = 0.005).
#>   Verdict: CELL - Attributable to a single cell 
#>     cell (1,2): reported 0.56, but given the other cells (within rounding) it must lie in [-0.884, -0.243];
#>     required edit -0.798, far exceeding the 0.005 rounding tolerance.
#>   Structural: Substantive: no correlation matrix lies within rounding of the reported values (smallest correction is ~42x the rounding step), so it is not explained by rounding of a structural dependency. 
#>   Severity: largest single edit needed 0.209 (egregious, rounding cannot excuse it; delta = 0.005);
#>             total mass (Frobenius) 0.392; best achievable lambda_min -0.46.
#>   - Single sole culprit corroborated by the sparse correction and an impossible triple. 
#>   - R^2 localizer: variable(s) {1, 2} are over-determined by the others (R^2 > 100%) -- conditional on the other cells being correct.

A joint fault (no single cell suffices)

Two independent inconsistencies mean no single edit repairs the matrix:

blk <- matrix(c(1, 0.9, 0.9, 0.9, 1, -0.9, 0.9, -0.9, 1), 3, 3)
R_joint <- matrix(0, 6, 6); R_joint[1:3, 1:3] <- blk; R_joint[4:6, 4:6] <- blk
diag(R_joint) <- 1
localize_psd_fault(R_joint, decimals = 2)$localization_verdict
#> [1] "joint"

The raw evidence

Every verdict carries the full box-aware evidence behind it — per-cell implied intervals (A), rounding-robust impossible triples (B), leave-one-out (C), the sparse minimal correction (D), and the R² / VIF variable localizer. Get it with fault_evidence():

ev <- fault_evidence(localize_psd_fault(R_bad, decimals = 2))
names(ev)
#> [1] "sole_culprit_cells" "impossible_triples" "lofo_restoring"    
#> [4] "sparse_support"     "rsquared"

The impossible-triple scan (component B) is also exposed on its own; it is pure interval arithmetic and needs no solver:

impossible_triples(R_bad, decimals = 2)
#> # A tibble: 1 × 4
#>       i     j     k box_max_det
#>   <int> <int> <int>       <dbl>
#> 1     1     2     3       -2.84

Benign causes come first

Non-PSDness is often benign. A composite score plus its subscores, or a full set of category dummies, is exactly rank-deficient by construction — not a sign of a data problem (van Tilburg & van Tilburg 2023; Lorenzo-Seva & Ferrando 2021). For a forensic tool this matters, so every impossible result carries a structural diagnosis that separates a benign near-boundary rank-deficiency from a substantive over-the-boundary violation.

Here a composite equal to the sum of three subscores is nudged just past the rounding boundary. The cellular verdict is diffuse (there is no single cell to blame), but the structural read names the likely cause:

Sig <- matrix(0.5, 3, 3); diag(Sig) <- 1
a <- c(1, 1, 1); r <- as.numeric(Sig %*% a) / sqrt(sum(a * (Sig %*% a)))
R_comp <- rbind(cbind(Sig, r), c(r, 1))
for (k in 1:3) R_comp[k, 4] <- R_comp[4, k] <- r[k] + 0.02   # nudge past the boundary
lf <- localize_psd_fault(round(R_comp, 2), decimals = 2)
lf$structural$severity_class
#> [1] "near-boundary"
lf$structural$pattern
#> [1] "composite"

Contrast this with the frustrated triangle, whose severity is far beyond rounding:

localize_psd_fault(R_bad, decimals = 2)$structural$severity_class
#> [1] "substantive"

Plausibility, even when possible

When a matrix passes the impossibility test it can still sit implausibly close to the boundary. For possible matrices the result reports a plausibility gradient: the per-variable reported-point R² and its closest approach to the 100% ceiling. The exactly-singular composite matrix (before the nudge) rounds to possible, but one variable’s R² exceeds 100% at face value:

R_exact <- round(rbind(cbind(Sig, r), c(r, 1)), 2)   # composite = sum of subscores
lf0 <- localize_psd_fault(R_exact, decimals = 2)
lf0$localization_verdict        # "none": possible given rounding
#> [1] "none"
lf0$plausibility$max_r2         # ... but a variable's reported-point R^2 exceeds 1
#> [1] 1.055

Implied intervals and imputation

implied_interval() reports the interval a cell could occupy for the matrix to be valid, given the others. Point it at a missing (NA) entry and it imputes the interval that entry must have occupied — a forensic “impute and check”:

R_miss <- round(rbind(cbind(Sig, r), c(r, 1)), 2)
R_miss[1, 4] <- R_miss[4, 1] <- NA        # pretend this correlation was unreported
implied_interval(R_miss)
#> # A tibble: 1 × 8
#>       i     j reported    lo    hi required_edit hold   status  
#>   <int> <int>    <dbl> <dbl> <dbl>         <dbl> <chr>  <chr>   
#> 1     1     4       NA 0.284 0.809            NA points feasible

A note on the engine

Everything above uses one self-contained numerical engine: a verified witness-vector bound and a one-sided Cholesky test (Rump 2006) for the rigorous sides, projections onto convex sets (POCS) as the search, and interval arithmetic for the triple scan. There is no dependency on an external SDP solver. See the package README for the full method provenance and references.
